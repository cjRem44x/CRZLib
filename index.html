Overview

CRZLib is a comprehensive utility library for Zig that simplifies common programming tasks. It provides a collection of well-tested, memory-safe functions for file operations, string manipulation, mathematical calculations, and more.

Memory Safe

All operations are designed with memory safety in mind, properly managing allocations and preventing leaks.

Performance Optimized

Uses fast algorithms and minimizes allocations for optimal performance in production environments.

Cross-Platform

Works across different platforms with special support for Windows-specific operations.

Easy to Use

Simple, intuitive API that follows Zig conventions and best practices.

Installation

Using Zig's Package Manager (Recommended)

Add CRZLib to your build.zig.zon:

.{
    .name = "my-project",
    .version = "0.1.0",
    .dependencies = .{
        .crzlib = .{
            .url = "https://github.com/cjRem44x/CRZLib/archive/refs/tags/v0.1.0.tar.gz",
            // Add the hash after first build attempt
        },
    },
}

Then add to your build.zig:

const crzlib_dep = b.dependency("crzlib", .{
    .target = target,
    .optimize = optimize,
});
exe.root_module.addImport("crzlib", crzlib_dep.module("crzlib"));

Manual Installation

Clone the repository:

git clone https://github.com/cjRem44x/CRZLib.git

Add to your build.zig:

const crzlib_module = b.addModule("crzlib", .{
    .root_source_file = b.path("path/to/CRZLib/src/crzlib.zig"),
});
exe.root_module.addImport("crzlib", crzlib_module);

File System Operations

read_file

pub fn read_file(allocator: std.mem.Allocator, path: []const u8) ![][]const u8

Reads a file and returns its contents as an array of strings (one per line).

Note: Memory must be freed by the caller using allocator.free() 
const std = @import("std");
const crz = @import("crzlib");

pub fn main() !void {
    const allocator = std.heap.page_allocator;
    const lines = try crz.read_file(allocator, "input.txt");
    defer allocator.free(lines);

    for (lines) |line| {
        crz.log(line);
    }
}

Complete Examples

File Processing Example

const std = @import("std");
const crz = @import("crzlib");

pub fn main() !void {
    const allocator = std.heap.page_allocator;

    // Read a file
    if (crz.is_file("input.txt")) {
        const lines = try crz.read_file(allocator, "input.txt");
        defer allocator.free(lines);

        crz.log("File contents:");
        for (lines) |line| {
            crz.log(line);
        }
    } else {
        crz.liberr("File not found!");
    }
}

String Manipulation Example

const std = @import("std");
const crz = @import("crzlib");

pub fn main() !void {
    const allocator = std.heap.page_allocator;

    // Concatenate strings

        crz.log(fruit);
    }
}

Interactive Console Application

const std = @import("std");
const crz = @import("crzlib");

pub fn main() !void {
    var buf: [256]u8 = undefined;

    crz.log("Welcome to CRZLib Demo!");

    const name = try crz.cin(&buf, "Enter your name: ");
    const greeting = try crz.strcat(std.heap.page_allocator, "Hello, ", name);
    defer std.heap.page_allocator.free(greeting);

    crz.log(greeting);

    const age_str = try crz.cin(&buf, "Enter your age: ");
    const age = crz.str_i32(age_str);

    crz.strout("You are ");
    crz.logn(age);
    crz.log(" years old!");
}

Error Handling

CRZLib follows Zig's error handling conventions:

  • All functions that can fail return error unions
  • Memory operations are safe and properly managed
  • File operations include proper error checking
  • String operations handle edge cases gracefully
  • Number parsing returns safe default values (0 or 0.0)
Best Practice: Always handle errors using Zig's try or catch keywords. Free allocated memory using defer to prevent leaks.

Performance Considerations

  • Mathematical functions use optimized algorithms (Newton-Raphson, Fast Inverse Square Root)
  • String operations minimize allocations where possible
  • File reading uses buffered I/O for efficiency
  • Random number generation uses cryptographically secure methods
  • All operations are designed for production use
